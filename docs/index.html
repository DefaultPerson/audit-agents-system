Publicly disclosed · June 5, 2026 Why this matters

A frontier model found a 4-year-old bug humans missed.

In late May 2026, Anthropic's Opus 4.8 — running an autonomous auditing agent on max effort — independently discovered a critical soundness bug in Zcash's Orchard shielded pool: a single missing copy-constraint in a halo2 ZK gadget that allowed undetectable in-pool counterfeiting. It was responsibly disclosed and fixed via an emergency soft fork and the NU6.2 hard fork. No exploitation was found.

If a frontier model can autonomously find a bug humans missed for four years in heavily-audited cryptography — the same autonomous-research method applies to the millions of unaudited, closed-source contracts on EVM chains.

This repository is a concept demonstration of that method — applied to EVM smart contracts. It is not a weapon, and it ships without targets or credentials.

The method

Five stages, from raw bytecode to a validated finding.

Every stage feeds the next an immutable artifact. Nothing reaches a report without evidence behind it.

01

Chain snapshot discovery + triage

Snapshot a chain at a pinned block, then triage for the targets worth researching: closed-source, high-value, old contracts — filtered by balance, proxy pattern, and verification status.

02

Decompile bytecode Dedaub + fallback

Decompile via Dedaub, with a bytecode / disassembly / storage-layout fallback layer for the cases it can't recover.

// Decompiled names are hints, not facts.

03

Immutable artifact bundle evidence base

Build a frozen bundle of ground truth: selectors, storage layout, recent traces and balances. Every later hypothesis must point back to something in here.

04

Autoresearch loop Researcher ⇄ Skeptic

Two different LLMs argue across specialist domains. A deterministic consensus gate passes a hypothesis only when it carries real evidence — selector, preconditions, impact, validation method, supporting fact.

05

Validators evidence-gated

Foundry pinned-fork test → ItyFuzz → symbolic (Mythril / Halmos) → economic checks. Only validated findings become an internal report.

// Disclosure is manual — never auto-sent.

The star of the show

The debate: Researcher ⇄ Skeptic, behind a consensus gate.

One model proposes attacks. A model from a different family attacks the proposal. Only evidence opens the gate.

auth / upgradeability proxy / storage / delegatecall accounting / share-math oracle / price / liquidity state-machine / lifecycle external-calls / reentrancy
Researcher →selector 0x3659cfe6 looks like an unguarded upgrade path on the proxy.
Skeptic →Show the missing access-control check and a storage fact that proves the slot is writable.
Gate ✓Evidence complete: selector · precondition · impact · validation method · cheap fact → validate on fork.
The gate is deterministic. selector preconditions expected impact validation method supporting cheap fact

All five, or the hypothesis goes back to the loop with a recorded reject reason — it never reaches a validator, and never reaches a report.

The key thesis

The loop doesn't care what it's reading.

It is chain-agnostic and domain-agnostic. The same Researcher ⇄ Skeptic method runs over EVM bytecode — or, as Zcash showed, a ZK circuit.

Chain-agnostic

EVM adapters share one contract — chain id, RPC, explorer metadata, native currency. Adding a chain is a config, not a rewrite.

Ethereum BSC Base Polygon Arbitrum + any EVM

Domain-agnostic

The debate operates on whatever ground truth you hand it. Swap the artifact layer and the same loop reasons about different material.

[evm]EVM bytecode & storage — millions of unaudited contracts
[zk]ZK circuits & gadgets — the Zcash / halo2 class of bug
[…]any artifact with selectors, state, and an oracle of correctness

The Orchard bug and an EVM proxy hijack are the same shape to the loop: a hypothesis, a challenge, and an evidence gate.

Architecture

End to end, the data flow.

Reasoning-only findings die in the loop. Only fork / fuzz / symbolic / economic evidence survives into a report — and disclosure stays in human hands.

Research & educational concept

This repository demonstrates a methodology — it is not a turnkey tool. It ships without credentials or live target lists.

Use only on testnet, or on contracts you are explicitly authorized to assess.
Findings are disclosed responsibly and manually — the pipeline never auto-sends.
High and critical findings require fork / fuzz / symbolic evidence before they exist as findings at all.
