// an open question – and one conjecture about the answer

How are so many bugs being found and drained this fast?

Crypto theft has surged – ~$3.4B in 2025, with Truebit, Balancer and Solv each draining tens to hundreds of millions in the last few months. The targets keep looking the same: old, closed-source, never-audited contracts. So how does anyone find these bugs at scale, before the defenders do? This page lays out one plausible method – automated, AI-driven vulnerability research – as an interactive schematic. The diagrams are live: click a stage, step the loop, flip the gate.

[Speculation] This is a concept demonstration of how it could be done, not an accusation of any specific attacker and not a product. That the capability is real is no longer hypothetical: in May 2026 Anthropic's Opus 4.8, driving an autonomous auditing agent, found a 4-year-old soundness bug in Zcash's Orchard shielded pool. Point the same method at the millions of unaudited, closed-source contracts on EVM chains and the question above starts to answer itself.

4 the validator ladder

Only a reproducer escalates

A passing consensus hypothesis is just a candidate. It climbs an escalation ladder; a stage promotes it only if the run emits the evidence marker. No marker, no finding.

1
Foundry pinned-fork test forge test

Replays a concrete tx sequence against real chain state at the pinned block. The default, cheapest reproducer.

promote onVALIDATED_EVIDENCE: true
2
ItyFuzz on-chain fuzzing

Searches for a value-moving sequence the fork test missed – stateful, on the same forked state.

promote onVALIDATED_EVIDENCE: true
3
Symbolic Mythril / Manticore

Solves path constraints for the preconditions and produces a concrete counterexample tx sequence, replayed on the fork.

requires concretecounterexample
4
Property Halmos / Echidna / Medusa

Checks an explicit invariant or postcondition with a shrunk counterexample and reproducible harness.

requires shrunkcounterexample
5
Economic checks profit-after-gas

Profit after gas, liquidity, slippage, manipulation cost. Never promotes alone – a fork/fuzz/symbolic reproducer must show the impact first.

gates positiveexpected profit
Validated findings become an internal report. Disclosure is manual – the pipeline never auto-sends. "a passing empty test is not evidence."

5 grounding

Why the question is real: classes that keep paying out

The six loop domains aren't abstract – each maps to a class that has drained real funds from deployed, often unaudited contracts in the last few months. These are the bugs a method like the one above would be hunting for.

Truebit Protocol~$26M · 8,535 ETH
Jan 2026 · the running example

Integer overflow in a bonding-curve getPurchasePrice returned a zero price for a crafted mint → free mint → reserves drained. The contract was ~5 years old, unverified, and unaudited – exactly the target profile a method like this would prioritize.

accounting / share-math ← overflow → zero price → free mint

The full data flow
